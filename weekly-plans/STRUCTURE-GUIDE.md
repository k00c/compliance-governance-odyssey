# Weekly Plan Structure Guide

## Overview

This document explains the improved day-by-day structure for weekly learning plans in this repository. The new structure is designed to make it easier to work through weekly plans without scrolling around to find information.

---

## Key Improvements

### Before: Scattered Information
The previous structure had:
- Daily tasks in one section
- All readings at the bottom in a separate "Readings & Resources" section
- Course information in a separate "Courses" section
- Projects described in a separate "Practical Projects" section
- Required lots of scrolling to find what you need for each day

### After: Self-Contained Days
The new structure has:
- **Each day is completely self-contained**
- Everything you need for that day is in that day's section
- Readings, resources, tasks, courses, and deliverables all in one place
- Work through the week day-by-day without scrolling

---

## Structure Benefits

### 1. **No Scrolling Required**
You can focus on one day at a time without jumping between sections. Open the file, find today's section, and you have everything you need.

### 2. **Clear Daily Goals**
Each day has:
- Specific time allocation
- Clear task checklist
- Readings with estimated times
- Deliverable with exact save location
- Reflection prompts

### 3. **Progressive Learning**
Days build on each other logically:
- Day 1-2: Foundational concepts and theory
- Day 3-4: Deeper understanding and practice
- Day 5-6: Hands-on projects and applications
- Day 7: Review, reflection, and preparation

### 4. **Portfolio Focus**
Each week produces concrete deliverables that build your portfolio for job applications.

---

## How to Use the New Structure

### Step 1: Start with Overview
Read the Overview and Learning Objectives to understand the week's goals.

### Step 2: Work Day by Day
1. Open the markdown file
2. Scroll to today's day (e.g., "Day 3")
3. Read through the entire day's section
4. Check off tasks as you complete them
5. Save deliverables to the specified location
6. Move to the next day when ready

### Step 3: Track Progress
- Use the checkboxes to track completion: `- [ ]` becomes `- [x]`
- Save deliverables to the specified folders
- Keep notes in the designated locations

### Step 4: Weekly Review
On Day 7, review all your work and reflect on the week's learning.

## Daily Section Components

Each day includes these components (as applicable):

### Required Components
- **Time Allocation:** How long the day's work should take
- **Tasks:** Checklist of things to do
- **Deliverable for Today:** What to produce and where to save it

### When Applicable
- **Readings for Today:** Table with resources, type, time, and links
- **Course Module for Today:** Specific modules to complete
- **Resources for Today:** Links and references
- **Practice Exercise:** Hands-on activities
- **Key Concepts to Master Today:** Important definitions and ideas
- **Reflection Prompts:** Questions to consider

---

## Example Day Structure

```markdown
## Day 3: Risk Assessment and Analysis

**Time Allocation:** 3-4 hours

### Tasks
- [ ] Understand qualitative vs quantitative risk assessment
- [ ] Learn likelihood and consequence rating scales
- [ ] Study risk matrices and heat maps
- [ ] Practice calculating risk ratings

### Readings for Today
| Resource | Type | Time | Link/Location |
|----------|------|------|---------------|
| Risk Assessment Methods | Article | 1 hr | Online search |
| Risk Matrix Guide | Reference | 45 mins | Various sources |

### Course Module for Today
- [ ] Module 3: Risk Assessment Basics (60 mins)

### Practice Exercise
Create a 5×5 risk matrix and plot 5 sample risks on it.

### Key Concepts to Master Today
- **Qualitative Assessment:** Uses descriptive scales
- **Quantitative Assessment:** Uses numerical values
- **Risk Matrix:** Visual tool showing likelihood vs consequence

### Deliverable for Today
✅ Completed risk matrix template saved to `artifacts/risk-registers/day-03-risk-matrix.xlsx`

### Reflection Prompts
- Which assessment method seems most practical?
- How confident do you feel creating risk matrices?
```

## Creating New Weekly Plans

When creating new weekly plans, follow the template in the Copilot Instructions:

1. **Start with the template** in `.github/copilot-instructions.md`
2. **Make each day self-contained** - include all readings, resources, and instructions
3. **Use consistent formatting** - follow the markdown structure
4. **Include clear deliverables** - specify exactly what to create and where to save it
5. **Add reflection prompts** - help learners process their learning
6. **Link to resources** - provide direct links where possible

## Tips for Working Through Weekly Plans

### Time Management
- Block out dedicated time for each day's work
- Don't rush—take the estimated time seriously
- If you need more time, that's okay—adjust your schedule

### Active Learning
- Don't just read—create the deliverables
- Use the checklists to track progress
- Complete reflection prompts to deepen understanding

### Portfolio Building
- Save all deliverables in the specified locations
- Keep your work organized for later use
- These artifacts demonstrate skills to employers

### Flexibility
- You can adjust the pace to fit your schedule
- If a topic needs more time, spread it over more days
- The structure is a guide, not a rigid requirement

## Comparison: Old vs New Structure

### Old Structure Example
```
Daily Schedule
  Day 1: Introduction (see tasks below)
  Day 2: Risk Identification
  ...

Readings & Resources (all at bottom)
  - Reading 1
  - Reading 2
  - Reading 3

Courses (separate section)
  - Module 1
  - Module 2

Projects (separate section)
  - Project 1 details
  - Project 2 details
```

**Problem:** Need to scroll between sections to see what to read, which course modules to complete, and what projects to work on.

### New Structure Example
```
Day 1: Introduction
  Tasks: [specific to Day 1]
  Readings for Today: [Day 1 readings only]
  Key Concepts: [Day 1 concepts]
  Deliverable: [Day 1 deliverable]

Day 2: Risk Identification
  Tasks: [specific to Day 2]
  Readings for Today: [Day 2 readings only]
  Course Module: [Day 2 modules]
  Deliverable: [Day 2 deliverable]
```

**Benefit:** Everything for each day is in one place. No scrolling required.
